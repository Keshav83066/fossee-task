# FOSSEE Workshop Booking - UI/UX Enhancement

This project is a modern, responsive redesign of the FOSSEE Workshop Booking system. Built with **React** and **Tailwind CSS**, the goal was to create a seamless experience for students, especially on mobile devices.

## 🚀 Improvements Made
- **Modern UI:** Replaced the basic HTML structure with a clean, card-based layout using a professional Blue and Slate color palette.
- **Mobile First:** Optimized inputs and buttons for easy touch interaction on small screens using Tailwind CSS.
- **Visual Hierarchy:** Used bold typography and color accents to guide the user towards the booking action.
- **Real-time Feedback:** Added interactive states for buttons and form inputs (focus rings and hover effects).

## 🧠 Reasoning (Required Questions)

### 1. What design principles guided your improvements?
I followed the principles of **Minimalism** and **Consistency**. By removing unnecessary clutter and using a consistent color palette, I ensured the user focuses solely on the booking form. I also used white space to make the interface feel light and modern.

### 2. How did you ensure responsiveness across devices?
I used a **Mobile-First approach** with Tailwind CSS utility classes. I ensured that on mobile devices, the form occupies the full width for better readability, and on larger screens, it stays centered in a neat card layout.

### 3. What trade-offs did you make between the design and performance?
I decided to use **standard system fonts and CSS-based styles** instead of heavy images or complex animations. This ensures that the website looks high-end but still loads instantly even on slow mobile networks (2G/3G).

### 4. What was the most challenging part of the task and how did you approach it?
Coming from a **Python background**, the biggest challenge was learning **React's JSX and Tailwind v4 configuration** in a very short time. I approached this by breaking the UI into smaller components and following documentation to fix PostCSS compatibility issues.
